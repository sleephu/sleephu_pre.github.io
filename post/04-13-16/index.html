Inner Class

  1. inner class is part of outer class, can access outer class’s private(or any other) members (one of key benefits, special relationship with outer class object)

    eg. class MyOuter { class MyInner {} }

  2. Instantiate inner class:

    a. inside outer class: MyInner inner = new MyInner();

    b. outside outer class: MyOuter.MyInner inner = MyOuter.new MyInner();

  3. method local inner class: class defined inside method; it cannot access method local variable, unless it’s marked as final, the only modifiers can apply to are abstract and final(unlike regular inner class, the modifiers are public, private, protected, final, abstract, etc.)

  4. Anonymous Inner class

    eg:

     Popcorn p = new Popcorn() {
    public void pop() {
      System.out.println("anoymous popcorn");
    }
 };

  5. Static Nested Class: is not inner class, is a static member of the enclosing class, like static method, it cannot access instance variables and nonstatic members of outer class eg:

    class BigOuter {
    static vlass Nested {}
 }

Thread

  1. Define Thread

    a. Extend java.lang.Thread (limitation: can’t extend anything else)

    b. Implement Runnable interface (benefit: can extend any class)

    eg:
   class MyRunnable implements Runnable {
      public void run() {System.out.println("implement job running in My Runnable");}
   }

  2. Instantiate thread

    a. Thread t = new Thread();

    b. need two classes

    i. Thread class: thread-specific code
ii. Runnable implementation class: job that should be run by a thread code
eg:
  i. instantiate runnable class
     MyRunnable r = new MyRunnable();
  ii. still need an instance of java.lang.Thread
     Thread t = new Thread(r); //pass Runnable to thread

  3. It’s legal the Thread itself implements Runnable

    eg:

    Thread t = new Thread(new MyThread());

  4. A thread is dead after run() method completes, the isAlive() is the best way to determine that; debug: getState()

  5. Starting thread: t.start(), only start once

    process: start –> go to runnable pool –> jvm select –> running

  6. which thread is running? Thread.currentThread().getName()

  7. Nothing is guaranteed, except: each thread will start, and each thread will run to completion

  8. which thread run first?

    There’s the Thread Scheduler, which is part of JVM, could schedule thread to run based on thread priority(1-10, bigger, higher), but not guaranteed. eg: t.setPriority(8)

  9. yield(): make currently running thread head back and make other with same priority get their turn

  10. join(): join at the end of the current running thread

  11. Thread problem: two different threads can access to a single instance of class?(multiple threads access to the same resource)

    eg.

    Two people each have checkbook, access to a single checking account

    1. Check the balance

    2. if have enough money, make the withdrawal

    It’s race condition, may produce corrupted data.

    Way to Protect: automic operation

     i. make variable private
 ii. Synchronize the code that modifies the variable

  12. Thread-safe class: class synchronized to protect its data

  13. wait(), notify(), notifyAll() are all instance methods of Object

  14. wait(): make thread go to waiting pool, if is current, gives up lock immediately

  15. notify(): notify certain thread in the waiting pool to get ready to get the lock(lock may be not available immediately)
