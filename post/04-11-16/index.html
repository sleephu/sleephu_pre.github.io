Operators

  1. Compound operator: the expression on the right side of = will always evaluated first.

    x *= 2 + 5; // x = x * (2 + 5);

  2. String concatenation:

     String a = "String";
 int b = 3;
 int c = 7;
 System.out.println(a + b + c); // output: String37
 System.out.println(a + (b + c)); //output: String10

  3. Variables marked final cannot be incremented or decremented

  4. ^ operator(logical XOR): returns true if exactly one operand is true

  5. Increment & Decrement

      int x = 2; int y = 3;
  if ((y == x++) | (x < ++y)) {
     System.out.println("x="+x+", y="+y);// output: x = 3, y = 4
  }
  a. y == x++ --> false: compare y == x first, then x+1 -> x = 3
  b. x < ++y --> true: ++y first -> y = 4, then compare x < y ? -> 3 < 4 true
  c. a | b --> print

Flow Control, Exception & Assertions

  1. Switch case:

    eg:
   switch(new Integer(4)) {
      case 4: System.out.println("boxing is ok"); // it is legal
    }

  2. if break is omitted, the program keeps executing the remaining cases blocks until break found or switch ends

  3. default case: use when none of case matched; it doesn’t have to come at the end of switch

  4. for loop

      eg: 
  a. for (initialization; condition; iteration) {}
  b. none of the tree sections required 
  for (;;) {System.out.println("inside an endless loop"); }
  c. don't need to increment or set anything, can put in virtually any arbitrary code statements you want to happend with each iteration of the loop
     int b = 3;
     for (int a = 1; b != 1; System.out.println("iterate")) {
         b = b - a;
     }
      output: iterate
              iterate

  5. break v.s continue

    break: stop execution of the innermost loop, start the next line of code after block

    continue: cease current execution, start next iteration of same loop if condition is still met

  6. Handling Exceptions

    a. code

        try {
     //some code, if put clearnup code here, it's easy to forget and redundent
    } catch(Exception e) {


    } finally {
      //clean up; release any system resource we allocate in the try clause
     // code here is always runs even exception not caught/try not run, unless System.exit() issue in try or catch clause
    }
    it's legal to omit either the catch clause or the finally clause, but not both

    b. Ducked exception: method call stack

    c. exceptions are always subclass of Java.lang.Exception

    two types:

      Unchecked Exceptions: Error(Things that not normarlly happend during program execution (eg. JVM run out of memory)), RuntimeException
  Checked Exception

    d. resist the temptation to write a single catch all exception handler

    eg:

       try {
   // some code
   } catch (Exception e) {
     e.printStackTrace();
     }

    // it defeats design objective, lose valuable information, reduce reliability of program since it’s likely an exception will be caught that the handler not know how to handle

    e. looking available catch clause from top down

    eg:

        try{
      ...
    }catch (FileNotFoundException fe) { 
      ... 
    }catch (IOException ie) {
      ...
    }

    f. Handle or Declare: each method must either handle all checked exceptions(which is excluding Error and RuntimeException) by supplying a catch clause or list

    g. some examples for Error and RuntimeException

     Error: StackOverflow (OS run out of space to store call stack)
 RuntimeException: NullpointerException, ArrayOutOfIndex

String, I/O, Formatting and Parsing

  1. String: immutable –> once assign a value, the value can never change; use once, remain in memory forever

      eg: 
  String x = "Java";
  x.concat(" Rules!");
  System.out.println(x); //output: Java

  2. Specific case analysis

      eg:
     String s1 = "spring"; //"spring" object referenced by s1
     String s2 = s1 + " summer";// " summer" object, lost; spring summer object, referenced by s2
     s1.contact("fall "); // "fall " object, lost; "fall spring" object, lost
     s2.concat(s1); //"spring summer spring" object, lost
     s1 += " winter"; //"winter" object, lost; "spring winter" object, referenced by s1, at the same time, "spring" object lost reference
     System.out.println(s1 + "  "+ s2);// output: spring winter spring summer

  3. String disanvertange: occupy large amount of memory, so JVM sets aside a special area of memory(String constant pool). When compiler encounters a string literal, it will check if the string literal in String constant pool or not first, if a match is found, no new objects created

  4. String v.s StringBuffer/StringBuilder

    String: immutable

    StringBuffer/StringBuilder: changable, use when you have to make a lot of modifications to Strings of characters; common use: file I/O, handle large block of data, pass it on, and then reuse the same memory to have the next block of data

  5. StringBuffer v.s StringBuilder

    StringBuffer: thread-safe, synchronization

    StringBuilder: faster, non-synchronization
