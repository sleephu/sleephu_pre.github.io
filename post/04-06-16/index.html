Declaration & Access Modifiers

  1. default v.s protected

    default: package level access proceted: package + kids (subclass, access through inheritance, but not through a reference to an instance of the superclass)

  2. final abstract —- wrong mix, opposite meaning

    final: cannot be subclassed; abstract: force to be subclassed

  3. Reason to use final: security(satefy), java core libraries (String, Thread.isAlive)

  4. abstract v.s interface

    a. All methods in interface is implicitly public and abstract

    b. Interface can only define constants(public static final), not instance variables, while in abstract class, it could have instance variables

    c. An interface can extend one or more interfaces, abstract could only extend one other abstract class

    d. An interface can’t extend anything but another interface

    e. An interface can’t implement another interface/class

    f. An interface must be declared with the keyword interface, while it’s legal that not declare an abstract class with abstract keyword

  5. “this”: refers to currently executing object

  6. final argument: keep the same value that the parameter had when it was passed into the method

  7. the first concrete class must implement all abstract methods of the super class

  8. synchronized: only applies to method; a method can be accessed by only one thread at a time

  9. static:

    a. exist independently of any instance created for the class

    b. static members exist before new instance of a class, and there will be only one copy of a static member regardless of the number of instances of that class

    c. all instances of a given class share the same value for any given static variable

  10. constructor:

    a. same name as class name

    b. no return type

    c. cannot be static (since it is associated with object instantiation)

    d. cannot be final/abstract (since cannot be overriden)

  11. enum: a special kind of class, it can be used to restrict a variable to have one of only a few pre-defined values, which is good to reduce bugs.

    a. defined in different places(inside or outside a class), different ways to access it.

    b. like other class, could have constructor, instance variables, methods, etc.

    c. constructor:

    I. can never invoke enum constructors directly, and the constructor is invoked automatically with arguments you define after the constant value
II. can define more than one arguments to the constructor (can overload)

    d. has static method: values(), returns array of the enum’s values

    eg:

     enum CoffeeSize {


   //4, 8, 10 are passed to the constructor
   SMALL(4), BIG(8), HUGE(10);
   CoffeeSize(int ounces) {
     this.ounces = ounces;
    }
    private int ounces;
    public int getOunces() {
       return ounces;
         } 
  }
