Generics and Collections

  1. Hashing retrieval:

    a. find the right bucket(using hashCode())

    b. search the bucket for the right element(using equals())

  2. equals() v.s ==

    equals(): object content meaningfully equal (the two object reference refer to the same object)

    ==: bits in the variables are identical

  3. collection v.s Collection v.s Collections

    a. collection: represent any of the data structure in which objects are sorted and iterated over

    b. Collection: java.util.Collection interface from which Set, List, Queue extend

    c. Collections: java.util.Collections holds a pile of static utility methods for use with collections

  4. ordered v.s sorted

    a. ordered: iterate through the collection in a specific(non-random) order

    b. sorted: the order in the collection is according to some rule, associate with the property of the objects themselves

  5. List (a list of things)

    a. ArrayList: faster iteration, not good for insertion/deletion

    b. Vector: like slower ArrayList (synchronized, thread-safe)

    c. LinkedList: faster insertion/deletion, not good for iteration

  6. Set (Uniqueness, unique of things)

    a. HashSet: unsorted, unordered

    b. LinkedHashSet: ordered version of HashSet (in the order of element insertion)

    c. TreeSet: sorted, natural order by default, could construct with self-defined order by using comparable/comparator

  7. Map (unique identifier)

    a. HashMap: unsorted, unordered, allow one null key and multiple null values

    b. HashTable: synchronized HashMap, doesn’t allow anything that’s null

    c. LinkedHashMap: ordered(insertion order), compared to HashMap, faster iteration, bad add/remove

    d. TreeMap: Sorted Map(natural by default)

  8. Queue (things arranged by order)

    a. PriorityQueue: ordered by priority according to a comparator

    b. LinkedList: basic queues, FIFO

  9. See the following code:

     List<String> myList = new ArrayList<String>();

    Good things:

     a. follow OOP, coding into interface
 b. use generics; <Strin> --> type of collection

  10. Useful methods

    a. Collections.sort(anyCollectionToSort, comparator);

    b. Arrays.sort(anyArrayToSort, comparator);

    b. Arrays.asList()

  11. Generics benefit: type safe, dont need to cast, and won’t break other’s existing code

  12. The type of the variable declaration must match the type you pass to the actual object type, not a subtype of the decalared type

    eg:

     List<Animal> animals = new ArrayList<Dog>(); // wrong, could only add Dog, if add Cat, then put the wrong thing, then compiler error

    reason: Compiler and JVM behave differently, JVM doesn’t know about the type information, and it doesn’t have exception for generics (not like ArrayStoreException for Array)
