Object Oriented

  1. Key Characteristics

    a. Encapsulation: associate with hiding implementation details, the ability to make changes in your implementation without breaking the code of others who use your code

    b. Inheritance: used to promote code reuse; use polymophism (many forms)

    i. IS-A: extends/implements
ii. HAS-A: based on usage, not inheritance

    c. Polymorphism: many forms (see eg. as follows)

    class Dog extend Animal implements Animatable {}
Dog is:
     i. An Object
     ii. An Animal
     iii. A Dog
     iv. An Animatable

  2. Overriding v.s Overloading

    a. Overriding: if A extends B, A has the opportunity to override method in B (except final); Overloading: name reuse

    b. Overriding cannot change argument list; Overloading must change argument

    c. The overriding method can throw any unchecked (runtime) exception, regardless of whether the overriden method declare the exception, but can only throw fewer or narrower exceptions; the overloading can declare new or broader checked exceptions

    d. The return type must be the same as or a subtype of the return type declared in the original overriden superclass, while overloading method can change the return type

    e. The overriding method cannot have more restrict access modifier than original overriden method in superclass, while overloading method could have any access modifier

    f. The reference type (not the object type) determines which overloaded method is invoked, while the object type determines which overriden method is invoked!!!(important)

    eg: Overloading:

     class Animal {}
 class Horse extends Animal {}
 class UseAnimal {
     public void doStuff(Animal a) {
        System.out.println("Animal Version");
     }
     public void doStuff() {
        System.out.println("Horse Version");
 }
   } 
 UseAnimal ua = new UseAnimal();
 Animal a = new Horse();
 ua.doStuff(a); // output is "Animal Version"
 ---------------------------------------------------------
 Overriding:
 suppose eat() in Animal prints out "Animal eat", in Horse prints out "Horse eat"
 Animal a = new Horse();
 System.out.println(a.eat()); //output: "Horse eat"

    Explian: compiler checks the reference type which is Animal, but it’s JVM which looks for the object type decides which version to be invoked

  3. Casting a. downcast: casting down the inheritance tree to a more specific class

    if (animal instanceof Dog) {
   Dog d = (Dog)animal;
   d.playDead();
   }

    b. upcast: works implicitly

    Dog d = new Dog();
Animal a1 = d; //upcast ok with no explicit cast
Animal a2 = (Animal)d; //upcast ok with an explicit cast

  4. Interface

    abstract class can choose to implement any, all, or none of methods from any of the interface

  5. Constructor:

    a. Every class has constructor (abstract class’s constructor is invoked when its concrete subclass is instantiated)

    b. no return type

    c. first statement must be super(); or this(); (if none was there, compiler insert super() call)

    d. if not type constructor, compiler generate a no-argu constructor automatically

    e. constructor name must match the class, and it’s legal to have method with same name as class name

    f. interface cannot have constructors, since they are not part of inheritance tree

    g. constructors are never inherited.(the type of constructor in super class can never determines the default constructor you get, and the default constructor is ALWAYS no-argu constructor)

    h. can overload constructor, and it’s used to provide alternate ways for clients to instaitate objects of your class. (p.s not always make sense, eg. java.awt.Color: you have to provide color info in constructor)

    i. constructor could have any access modifier, including private(only code with the class itself can instantiate an object of that type, if the private constructor class wants to allow an instance of the class to be used, the class must provide static method or variable that allow access to an instance created from within the class)

  6. Static

    a. static = class; nonstatic = instance

    b. cannot access a nonstatic(instance) member(variables/methods) from static method

      class Animal {
   static void doStuff() {
       System.out.println("a");
      }
    }
  class Dog extends Animal {
 static void doStuff() {
     System.out.println("d");
     }  
   }
  public static void main(String[] args) {
     Animal[] a = {new Animal(), new Dog(), new Animal};
     for (int x = 0; x < a.length; x++) {
     a[x].doStuff(); // output: a a a
    }
 }

  7. Coupling v.s Cohesion

    a. Coupling: the degree to which one class knows about another class

    b. Cohesion: the degree to which a class has a single, well-defined row or reponsibility
